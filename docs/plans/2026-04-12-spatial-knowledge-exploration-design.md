## 1. The Core Loop: "Mine, Refine, Deploy"

Luna OS transforms mundane engineering tasks into a strategic game loop:
1.  **Mine (Memory)**: Navigate the 3D Nebula to identify knowledge gaps or data discrepancies.
2.  **Refine (Orchestration)**: Assemble a specialized "Party" of agents to tackle the mission.
3.  **Deploy (Execution)**: Watch the "Raid" in real-time via the HUD as agents interact with the graph and each other.

---

## 2. Gameplay Mechanics

### 2.1. The "Party" (Agents)
Agents are no longer just service workers; they are specialized heroes with class-based stats:
- **Triage (Tank)**: High-bandwidth, identifies the core threat (severity).
- **Investigator (Scout)**: High-speed, pulls observations from the deep nebula.
- **Analyst (Mage)**: High-compute, transforms raw data into root causes.
- **Commander (Paladin)**: High-synthesis, produces the final "Loot" (report/fix).

**Stats tracked in HUD**:
- **Mana (Tokens)**: The fuel for agent actions.
- **Heat (Cost)**: Cumulative USD spend of the current party.
- **XP (RL Weight)**: Real-time visual indicator of the agent "learning" from this mission.

### 2.2. The "Missions" (Workflows)
Every `CoalitionWorkflow` is a **Raid**. 
- **Objectives**: Mapped from the Temporal workflow steps.
- **Bosses**: Complex contradictions or schema mismatches in the Knowledge Graph that require high consensus to "defeat."
- **Loot**: New `KnowledgeEntities` and `BlackboardEntries` generated by the mission.

### 2.3. The "Open World" (Memory)
The Knowledge Graph is the map. 
- **Biomes**: Clusters of related data (e.g., "The SAP S4 Biome", "The B2C Hybris Sector").
- **Fog of War**: Areas of the graph with low confidence or stale embeddings appear dim until "scanned" by an agent mission.

## 4. Native Webcam & Tracking Pipeline

To achieve "Tony Stark" latency, the tracking pipeline is split between the native Rust backend and the 3D frontend:

1.  **Native Capture (Rust)**: Tauri accesses the webcam using the `nannou_video` or `opencv-rust` crate to bypass browser bottlenecks.
2.  **Tracking (MediaPipe/ONNX)**: Hand landmarks are processed via a lightweight ONNX model running natively in a background Rust thread (`tokio`).
3.  **Event Stream**: Normalized 3D coordinates (X, Y, Z) and Gesture States (PINCH, FIST, POINT) are emitted to the React frontend at 60fps via Tauri's `Window::emit`.
4.  **Calibration**: HUD includes a "Sync" animation where the user aligns their hand with a 3D overlay to calibrate depth sensitivity.

## 5. Implementation Phases

### Phase 1: The Transparent Window
- [ ] Configure `tauri.conf.json` for multi-window support.
- [ ] Create `spatial_hud` window with `transparent: true` and `decorations: false`.
- [ ] Implement Rust-side window shadowing and "click-through" logic (`set_ignore_cursor_events`).

### Phase 2: Rust UMAP Engine
- [ ] Integrate `umap-rs` into the Tauri backend.
- [ ] Create a Tauri command `get_spatial_data` that fetches embeddings from the API and projects them in a background Rust thread.
- [ ] Implement caching for 3D coordinates on the local machine for instant loading.

### Phase 3: Native Gesture Processing
- [ ] Implement a Rust-based webcam listener.
- [ ] Integrate a lightweight gesture model (e.g., a port of MediaPipe or a custom ONNX model) to run natively.
- [ ] Stream gesture events (X, Y, Z + GestureType) to the frontend via Tauri's `emit` system.
