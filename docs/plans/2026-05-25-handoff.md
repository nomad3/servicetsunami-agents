## §1 — The agent_token tier=anonymous incident (resolved)

### The breach pattern Luna surfaced 2026-05-24

Luna's WhatsApp dispatch_agent attempt the night before reported *"the live MCP agent registry is still rejecting this chat session as tier=anonymous, so I could not create them directly through the live MCP tool."* The 5-fleet-agents work in PR #712 had to ship via shell-only workspace writes (not MCP) because of this.

### What we thought it was (PR #713's plan, written 2026-05-24 night)

A propagation gap: agent_token JWT is minted at chat dispatch but not threaded into each subprocess CLI's MCP-client env. The plan scoped 5 phases including per-CLI MCP config rewriters for codex/claude_code/gemini_cli/copilot_cli/opencode.

### What it actually was (Phase 0 audit, 2026-05-25 morning)

The mint→header leg was already correct since PR #692. Codex emits `http_headers` correctly in TOML. claude_code writes the mcp_config JSON verbatim. The headers WERE arriving at mcp-tools.

The bug: `apps/mcp-server/src/mcp_auth.py::_get_header()` was reading `getattr(rc, 'headers', None)`. But FastMCP's RequestContext doesn't expose `.headers` directly — the Starlette Request lives at `rc.request`, headers at `rc.request.headers`. This was wrong since (probably) the file was first written.

### The fix (PR #716, merged `3bed2c11`)

One-attribute hop in `_get_header()`. Added the primary path `rc.request.headers`; preserved all 3 legacy fallback branches for backwards compat. 20/20 unit tests pass.

### Verification

Post-deploy:
- ~2000/24h SHADOW `tier=anonymous` denials → 2 in 15 min, both `tier=tenant_header` (not anonymous)
- Simon's tenant: zero denials in observation window
- Successful tool_audit lines for `list_calendar_events` etc. now landing without SHADOW flag

## §2 — The opencode persona-leak (resolved)

### The bug

Luna's 2026-05-24 WhatsApp fallback to opencode produced *"I'm OpenCode, an AI coding assistant..."* instead of as Luna. Both opencode dispatch paths (`execute_opencode_chat` server + `_execute_opencode_chat_cli` fallback) did `prompt = task_input.message` directly, without prepending the agent's `instruction_md_content` (persona). codex + claude_code both do the prepend.

### The fix (PR #717, merged `c47f64f1`)

Mirror the codex pattern. Plus, per superpowers review's IMPORTANTs:
- Server-path coverage test (httpx.post mock asserts persona in `parts[0].text`)
- Reorder so persona sits at top + `[Context: tenant_id=...]` lands INSIDE the User Request block (closer to the message it scopes)

10/10 tests pass. Final shape:
```
<persona>
\n\n# User Request\n\n
[Context: tenant_id=...]\n\n
<message>
```

### Verification

In-container Python introspection confirmed both paths use new `user_block` composition. End-to-end behavior verification needs Luna to actually fall back to opencode (cloud quota out) — structural tests + ordering invariant cover the regression class.

## §6 — Working patterns established/reinforced today

1. **Phase 0 audit is load-bearing.** The agent_token plan would have shipped a multi-day per-CLI rollout that didn't fix the actual bug. The ~30-minute Phase 0 audit caught it. Lesson preserved in `phase1-resolution.md` §7.

2. **Diagnostic-first when uncertain.** F2 spent 3 PRs (v1 diagnostic → v2 diagnostic → fix) instead of guessing. Worth the deploy cycles because guessing would have left the same ambiguity (we tried `-U -A` first and got nothing; the diagnostic data is what made `delete-then-add` obvious).

3. **`-U` doesn't touch ACL.** Specific macOS Keychain gotcha to remember. PR #720 → #721 caught it.

4. **`(exit=$?)` after a pipe captures the LAST command's exit, not the pipeline's.** Bash semantics. Use `out=$(cmd); rc=$?` to capture the real one.

5. **macOS exit 36 from `security` = errSecInteractionNotAllowed.** Silent, no error printed to stderr. The probe pattern that revealed it: capture both exit code AND the output length AND inspect ACL directly.

6. **Plans live in `docs/plans/`** per `feedback_always_document_plans`. Today added 4 plan docs (`phase0-baseline`, `phase1-resolution`, this handoff, and the original `propagation` plan from 2026-05-24 night). The corpus has the full arc.
