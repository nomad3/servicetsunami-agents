## 1. Skills Marketplace v2 — **Shipped 2026-04-26** (PRs #182–#193)

File-based skill library laid out across two folders on a shared volume. Replaces the prior in-DB three-tier (native/community/custom) layout.

**Layout:**
```
skills/
├── _bundled/                 # read-only, ships with the container
│   └── <skill-slug>/SKILL.md (+ optional script.py / script.sh / prompt.md)
└── _tenant/<tenant-uuid>/    # custom + community-imported, per-tenant
    └── <skill-slug>/...
```

**Key changes:**
- Skill format normalized on **Claude-Code-style `SKILL.md`** (frontmatter + instructions). Engines: `python`, `shell`, `markdown`, `tool`.
- Migration **110** adds `library_revisions` table — every change to a skill is auditable.
- New MCP tools: `update_skill`, `update_agent`, `read_library_skill` for full CRUD from any CLI runtime.
- Code-worker reads skills via the **`read_library_skill` MCP tool** rather than mounting the library into the worker pod (intentional — keeps the worker stateless and the library the single source of truth).
- Removed: `match_skills_to_context` MCP tool (replaced by the auto-trigger pgvector flow), GitHub-import flow.

**Plan:** `docs/plans/2026-04-26-skills-fleet-alignment-plan.md`

## 2. External Agents + A2A v2 — **Shipped 2026-04-26** (PRs #194–#205)

Workflows are the audit + dispatch spine. **No new `agent_messages` table** — handoffs ride on existing chat + workflow infrastructure.

**Architecture:**
- Coalition / handoff dispatches resolve through `_call_agent`, which routes UUIDs to native or external agents transparently.
- External agents register via `external_agents` + `external_agent_adapter.py` (OpenAI Assistants, webhooks, MCP protocol, Microsoft Copilot Studio, Azure AI Foundry).
- Handoffs persist as `ChatMessage(context.kind="handoff")` + `WorkflowRun`. Auditing comes for free from the dynamic-workflow run history.
- Patterns ship as **`workflow_templates` JSON** (incident_investigation, deal_brief, cardiology_case_review).
- Reliability shim mirrors Temporal `RetryPolicy` for native + external calls.

**New endpoints:**
- `GET /agents/microsoft/discover` (#251) — enumerate Copilot Studio + AI Foundry agents available on the tenant's Microsoft account.
- `POST /agents/import` (#243) — adopt Microsoft Copilot Studio + AI Foundry agents on the **GitHub Copilot CLI runtime**.

**UI:**
- Unified **Hire wizard** (#205) for native + external + marketplace + import paths.
- Routing transparency footer (#256) under assistant messages — shows which CLI ran, which agent, why.

**Plan:** `docs/plans/2026-04-26-external-agents-and-a2a-enhancement-plan.md`
