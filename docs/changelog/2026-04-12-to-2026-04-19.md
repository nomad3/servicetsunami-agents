## 5. Luna Native Client — Voice + Avatar — **Shipped 2026-04-19** (PR #154)

- **Visual avatar**: `LunaAvatar` SVG reacts to emotional state (idle, thinking, happy, alert)
- **Native push-to-talk**: Rust audio capture via `cpal`. `start_audio_capture` returns `AudioConfig{sample_rate, channels}`. Stream is built AND dropped on the same spawned thread (CoreAudio-safe — `Stream` is `!Send` on macOS).
- **Global shortcut**: `Cmd+Shift+Space` holds PTT (emits `voice-start`/`voice-stop`). Unregistered on window destroy.
- **WAV header**: `useVoice.encodeWav()` wraps Float32 PCM in a proper RIFF/PCM16 container before posting to `/api/v1/media/transcribe` (raw PCM would fail transcription).
- **VoiceProvider context**: `useVoice` must be instantiated exactly once per app (via `VoiceContext`). `ChatInterface` and `CommandPalette` both consume the same context to avoid duplicate `audio-chunk` listeners.

**Phase 2 (same day, `e79bc35b`):** Device Bridge (`apps/device-bridge`), registry, camera integration.

## 7. Security Hardening — **Shipped 2026-04-18** (PR #151, #152)

Defaults removed for three critical secrets; startup fails if they're missing.

| Change | Commit |
|--------|--------|
| Remove default values for `SECRET_KEY`, `MCP_API_KEY`, `API_INTERNAL_KEY` | `bc390966` |
| `html.escape()` all user input in OAuth callback (reflected XSS fix) | `5ee2973e` |
| Require superuser to import GitHub skills (closes RCE path) | `0013ea42` |
| Run skill scripts in subprocess with sensitive env vars stripped | `02015780` |
| Password reset: SHA-256 hash tokens, `hmac.compare_digest`, no debug_token leak, identical response for existing vs missing email | `c1c4b2a6`, `661eb053` |
| Platform credentials added to sensitive env blocklist | `1dfe9caf` |
| Remove hardcoded `dev_mcp_key` from `docker-compose.yml` environment blocks (overrode rotated env_file values) | `8da8c35c` |

**Rate limiting (`slowapi`):** login 10/min, password recovery 3/hour, reset confirm 5/hour.
**Media upload hardening**: transcription streams to disk (prevents OOM DoS on large uploads).

**Key rotation footgun:** In `docker-compose.yml`, `environment:` values always override `env_file` values. After rotating a key, audit every `environment:` block for hardcoded overrides. Always recreate with `docker compose up -d --force-recreate <service>`.

**Reports:**
- `docs/report/2026-04-17-platform-security-audit.md` — initial findings
- `docs/report/2026-04-18-full-security-audit.md` — full 8-finding audit
- `docs/report/2026-04-18-pentest-verification.md` — black-hat verification of fixes
- `docs/plans/2026-04-18-security-remediation-plan.md` — 4 remaining open items

**Rejected fix (for awareness):** A `_is_malicious_prompt()` regex filter added by Gemini blocked common shell characters (`;`, `&`, `>`, `<`, `|`) in user messages — 80% false positive rate on normal language. Reverted (`11f43837`). Real protection comes from subprocess sandboxing of skill execution.

## 8. Cardiac Report Generator Workflow — **Shipped 2026-04-19**

First new native dynamic workflow template since 2026-04-03 (now 26 total).

**Flow (for HealthPets / Brett The Cardio Vet tenant):**
1. `search_emails` — find patient email from JUMBOmail (Gmail query input)
2. `read_email` — get message body + attachment IDs
3. `download_attachment` — extract PDF text content
4. `agent (luna)` — generate DACVIM-format cardiac evaluation report (History, Exam, Echo Measurements table, Assessment, Plan with ACVIM/HCM staging)
5. `create_drive_file` — save as Google Doc (mime_type = `application/vnd.google-apps.document`)

**Inputs:** `patient_name`, `visit_date`, `email_query`, `account_email`, `drive_folder_id`.

Prompt logic reuses the original ADK `generate_cardiac_report` tool (deleted 2026-03-18 with the ADK removal).
